CSC 300/Data Structures in Java I
Assignment: Search text in file

15 points

Setting up

First, please read the section "Assignments and grading" in the syllabus. Then, following the steps for creating programs, create a package called searchText.

Specifications

Write a program called SearchText.java that:

  1. Prompts the user for a text file pathname.
  2. Prompts the user for a search string.
  3. Reads in all of the text in the file.
  4. Splits that text into an array of words.
  5. Prints a list of every word that contains the search string.
  6. After that list, prints a count of how many words contain that search string.
For example, running this program for the file constitution.txt and using the search string qu produces the output:

As models, consult the programs from week 1 that count words. You will have to look up the method in the String class that indicates whether one string contains another.

Submission instructions

When done correctly, you will have a single program SearchText.java in a package called searchText. This is located in your file system under the workspace folder you created for Eclipse. Starting at that folder, navigate to algs4/src/searchText. Zip that folder (that is, zip the folder searchText NOT just the file SearchText.java) into a zip file called searchText.zip and submit that zip file into the D2L dropbox for this assignment no later than its due date.

Four very important points to remember:

  1. Your name must appear in a comment at the top of the program. You can insert it either in this style:

    or this style:

 */
  2. Give the files the names EXACTLY as specified, including the capitalization. Not doing so will cause points to be deducted.
  3. A submission may be graded as soon as it appears in its drop box. In other words, don't submit until you're sure that you want the program graded.
  4. Just after the due date, zero points will automatically be given any assignment not submitted.
